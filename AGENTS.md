# AGENTS.md

## 项目技术栈

- 前端框架：Vue 3
- 开发语言：JavaScript
- 后端服务：当前阶段无后台
- 数据存储：优先使用浏览器本地存储，除非后续需求另有说明

## 开发规则

- 开始开发前，先理解需求并给出解决方案。
- 用户确认方案后，再执行文件创建、修改或删除。
- 修改已有文件前，先查看文件内容和项目结构。
- 不随意修改与当前任务无关的文件。
- 新增功能时，优先遵循 Vue 3 + JavaScript 的实现方式。
- 完成开发后，需要进行基础验证。

## Git 提交规则

- 每次新增功能后，需要提交一次 Git commit。
- 每次修复 Bug 后，需要提交一次 Git commit。
- Git commit message 格式为：年月日+修改的内容。
- 日期格式使用 8 位数字：YYYYMMDD。
- 示例：20260710 新增任务管理页面
- 示例：20260710 修复任务筛选显示异常

## 文档规则

- 产品需求文档统一放在 `doc` 目录。
- 文档文件名尽量清晰，并包含模块名称和版本号。
- 需求变更较大时，优先新增版本文档，避免直接覆盖历史版本。

## 项目约定

- 当前项目优先按单页面前端应用推进。
- 无后台功能默认不引入服务端依赖。
- 如果后续需要引入依赖、调整目录结构或增加构建工具，应先说明方案并等待确认。
