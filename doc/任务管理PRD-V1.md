# 任务管理 PRD V1

## 1. 项目概述

本项目为一个单网页版本的 TodoList 任务管理工具，面向个人用户的轻量任务记录与管理场景。产品无需后台服务、无需登录注册，所有任务数据保存在用户当前浏览器本地。

## 2. 产品目标

- 帮助用户快速记录、查看、编辑和完成个人任务。
- 支持页面刷新后任务数据仍然保留。
- 提供简单清晰的任务筛选和统计能力。
- 保持单页面、低复杂度、易使用的交互体验。

## 3. 用户场景

### 3.1 快速记录任务

用户打开页面后，可以直接输入任务标题并新增任务，用于临时记录待办事项。

### 3.2 管理任务状态

用户完成某项任务后，可以将任务标记为已完成；如果误操作，也可以恢复为未完成。

### 3.3 回顾不同状态任务

用户可以在全部任务、未完成任务、已完成任务之间切换，快速聚焦当前需要处理的内容。

### 3.4 修改或删除任务

用户可以编辑已有任务内容，也可以删除不再需要的任务。

## 4. V1 功能范围

### 4.1 任务列表模块

#### 功能说明

- 展示当前浏览器本地保存的所有任务。
- 默认按创建时间倒序展示，最新创建的任务显示在前。
- 任务需要区分未完成和已完成状态。
- 当任务列表为空时，展示空状态提示。

#### 任务展示内容

- 任务标题
- 任务完成状态
- 创建时间
- 优先级
- 截止日期
- 操作入口：编辑、删除

### 4.2 新增任务模块

#### 功能说明

- 用户可以新增一条任务。
- 任务标题为必填项。
- 任务描述、截止日期、优先级为可选项。
- 支持点击新增按钮创建任务。
- 支持在任务标题输入框按回车创建任务。

#### 输入规则

- 任务标题不能为空。
- 任务标题去除前后空格后仍为空时，不允许新增。
- 新增成功后，清空输入区域。
- 新增任务默认状态为未完成。
- 新增任务默认优先级为普通。

### 4.3 任务编辑模块

#### 功能说明

- 用户可以编辑已有任务。
- 可编辑字段包括任务标题、任务描述、截止日期、优先级。
- 编辑时提供保存和取消操作。

#### 编辑规则

- 保存时任务标题不能为空。
- 点击取消后，不保存本次修改。
- 编辑成功后，任务列表立即更新。

### 4.4 任务状态模块

#### 功能说明

- 用户可以将任务标记为已完成。
- 用户可以将已完成任务恢复为未完成。
- 已完成任务需要有明显视觉区分。

#### 展示规则

- 已完成任务标题可显示删除线。
- 已完成任务整体颜色可弱化。
- 任务状态变化后，统计数据同步更新。

### 4.5 任务删除模块

#### 功能说明

- 用户可以删除单个任务。
- 用户可以一键清除所有已完成任务。

#### 删除规则

- 删除单个任务时，建议提供确认提示。
- 清除已完成任务时，必须提供确认提示。
- 删除后任务不可恢复。
- 删除后本地存储同步更新。

### 4.6 筛选模块

#### 功能说明

提供三种任务筛选方式：

- 全部任务
- 未完成任务
- 已完成任务

#### 筛选规则

- 默认显示全部任务。
- 切换筛选条件时，不改变任务原始数据。
- 当前筛选项需要有明确选中状态。

### 4.7 搜索模块

#### 功能说明

- 用户可以通过关键词搜索任务。
- 搜索范围包括任务标题和任务描述。

#### 搜索规则

- 搜索关键词为空时，展示当前筛选条件下的任务。
- 搜索与筛选可以同时生效。
- 搜索匹配不区分大小写。

### 4.8 排序模块

#### 功能说明

V1 支持以下排序方式：

- 按创建时间排序
- 按截止日期排序
- 按优先级排序

#### 排序规则

- 默认按创建时间倒序。
- 优先级排序规则为：高 > 普通 > 低。
- 无截止日期的任务在按截止日期排序时显示在后方。

### 4.9 本地存储模块

#### 功能说明

- 使用浏览器 localStorage 保存任务数据。
- 页面刷新或关闭后再次打开，任务数据仍然存在。
- 不需要后台数据库。

#### 存储规则

- 每次新增、编辑、删除、状态变更后，都需要同步写入 localStorage。
- 页面初始化时，从 localStorage 读取任务数据。
- 当 localStorage 中没有数据时，初始化为空任务列表。

### 4.10 统计模块

#### 功能说明

页面需要展示基础任务统计信息：

- 总任务数
- 未完成任务数
- 已完成任务数
- 完成率

#### 统计规则

- 统计基于全部任务数据，不受当前筛选条件影响。
- 完成率 = 已完成任务数 / 总任务数。
- 总任务数为 0 时，完成率显示为 0%。

### 4.11 页面交互与样式模块

#### 功能说明

- 页面为单网页应用。
- 支持桌面端和移动端浏览。
- 操作反馈清晰。
- 信息层级简单，避免复杂导航。

#### 页面建议结构

- 顶部：应用标题与任务统计
- 中部：新增任务输入区
- 工具区：搜索、筛选、排序
- 主体：任务列表
- 底部或工具区：清除已完成任务

## 5. 非功能需求

### 5.1 兼容性

- 支持主流现代浏览器。
- 支持桌面端和移动端基础适配。

### 5.2 性能

- 常规个人任务数量在 500 条以内时，页面操作应保持流畅。
- 新增、编辑、删除、筛选等操作应即时响应。

### 5.3 可用性

- 用户无需学习复杂操作即可完成任务管理。
- 关键操作按钮文字清晰。
- 删除等不可恢复操作需要确认。

### 5.4 数据安全

- 数据仅保存在用户浏览器本地。
- 不上传任何任务数据。
- 更换浏览器、清理浏览器数据或更换设备后，原数据不会自动同步。

## 6. 数据字段设计

单条任务数据建议包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 任务唯一 ID |
| title | string | 是 | 任务标题 |
| description | string | 否 | 任务描述 |
| completed | boolean | 是 | 是否已完成 |
| priority | string | 是 | 优先级：high、normal、low |
| dueDate | string | 否 | 截止日期 |
| createdAt | string | 是 | 创建时间 |
| updatedAt | string | 是 | 更新时间 |
| completedAt | string | 否 | 完成时间 |

## 7. 页面状态

### 7.1 空状态

当没有任何任务时，展示提示文案，例如：

> 还没有任务，先添加一个待办事项吧。

### 7.2 无搜索结果状态

当搜索或筛选后没有匹配任务时，展示提示文案，例如：

> 没有找到匹配的任务。

### 7.3 输入错误状态

当用户新增或保存任务时，标题为空，需要给出提示，例如：

> 任务标题不能为空。

## 8. 验收标准

- 用户可以成功新增一条任务。
- 用户新增空标题任务时，页面给出提示且不会创建任务。
- 用户可以查看全部任务列表。
- 用户可以编辑任务并保存修改。
- 用户可以取消编辑，取消后任务保持原内容。
- 用户可以删除单个任务。
- 用户可以标记任务为已完成。
- 用户可以将已完成任务恢复为未完成。
- 用户可以在全部、未完成、已完成之间筛选任务。
- 用户可以通过关键词搜索任务。
- 用户可以按创建时间、截止日期、优先级排序任务。
- 页面可以正确展示总任务数、未完成数、已完成数和完成率。
- 刷新页面后，已有任务仍然存在。
- 清除已完成任务前会出现确认提示。
- 页面在手机和电脑浏览器上均可正常使用。
